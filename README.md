# NeighbourLink Static Prototype

This repository contains the Stage 2 design prototype for **NeighbourLink**, a community ride-sharing web platform for one metropolitan council area.

The prototype is based on the Stage 1 PowerPoint concept. It focuses on page structure, navigation logic, and representative screens for the rider and driver workflows.

## How to Open

Open `index.html` in a web browser.

No setup is required.

## Prototype Constraints

This prototype follows the Stage 2 requirements:

- HTML and CSS only
- No backend
- No data storage
- No scripting
- Navigation and layout focus only
- Representative pages rather than a complete implemented system

## Pages Included

- `index.html` - Home page with separate rider and driver entry points
- `find.html` - Rider search page
- `results.html` - Available ride matches page
- `request.html` - Rider request details page
- `confirmation.html` - Request confirmation page
- `offer.html` - Driver ride offer page
- `driver.html` - Driver request review page
- `login.html` - Login mockup
- `signup.html` - Sign up mockup
- `help.html` - Help and scope summary page

## Main Navigation Flow

Rider flow:

1. Home
2. Find a Ride
3. Available Rides
4. Request Ride
5. Confirmation

Driver flow:

1. Home
2. Offer a Ride
3. Ride Requests
4. Accept or decline rider requests

## Design Notes

The visual style uses thick borders, simple forms, role cards, step indicators, and list-based ride/request screens to match the hand-drawn wireframe direction from the Stage 1 presentation.

The prototype intentionally excludes payment processing, live location tracking, route optimisation, emergency features, admin dashboards, chat, and multi-city scaling.
